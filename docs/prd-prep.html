PRD 작성 준비 문서 (PRD Preparation)

1. 프로젝트 개요

1.1 프로젝트 명

  • RocketGrowth Console (쿠팡 판매자 관리 콘솔)

1.2 프로젝트 배경

  • 쿠팡 판매자 사이트의 정보를 효율적으로 관리하기 위한 통합 관리 도구 필요
  • 쿠팡 API를 활용한 자동화된 데이터 수집 및 관리 시스템 구축

1.3 개발 방식

  • MVP (Minimum Viable Product) 방식으로 단계적 개발
  • 핵심 기능 우선 구현 후 점진적 확장

2. 문제 정의

2.1 현재 Pain Points (확인 필요)

  • [V] 여러 화면을 오가며 정보를 확인해야 하는 비효율성

  • [V] 주문 관리의 복잡성

  • 효율적인 재고 관리
  • 판매 현황 추적 및 분석
  • 주문 처리 프로세스 개선

5. 기술 스택

5.1 Frontend

  • Framework: React + TypeScript
  • Build Tool: Vite
  • UI Library: [V] Tailwind CSS + headlessui
  • State Management: [V] TanStack Query + useState/Context
  • Routing: [V] TanStack Router
  • Charts: [V] Recharts

선택 이유: - Tailwind: AI 에이전트 친화적, 빠른 프로토타이핑 - TanStack Query: 서버 상태 관리 전문 (캐싱, Optimistic Updates) - TanStack Router: 타입 안전 라우팅, 파일 기반 - Recharts: React 네이티브, 선언적 API

5.2 Backend

  • Language: [V] Go 1.22
  • Framework: [V] Echo
  • Database: [V] SQLite
  • SQL: [V] Plain SQL (ORM 사용 안 함)
  • Migration: [V] golang-migrate
  • Background Jobs: [V] Goroutines + Ticker
  • Cache: [V] In-Memory (sync.Map)
  • Authentication: JWT + OAuth

선택 이유 (Agentic Coding 최적화): - Go: 명시적 Context, 빠른 컴파일/테스트, AI 에이전트 친화적 - Echo: 단순하고 명확한 라우팅 - SQLite: 배포 간소화, MVP에 충분 - Plain SQL: AI가 SQL을 매우 잘 생성 - Goroutines: 복잡한 Task Queue(Celery) 불필요

5.3 External API

  • Coupang API
  • API 버전 확인 필요
  • 사용 가능한 엔드포인트 확인
  • Rate Limit 확인
  • API 키 발급 프로세스

5.4 Infrastructure

  • [V] 호스팅 환경: 로컬 컨테이너 (Docker Compose)
  • [V] Hot Reload: air (Backend), Vite HMR (Frontend)
  • [ ] CI/CD 파이프라인 (Phase 2)
  • [ ] 모니터링 도구 (Phase 2)

배포 전략: - MVP: Docker Compose로 로컬/VPS 배포 - Phase 2: PostgreSQL 마이그레이션 고려 (사용자 증가 시)
